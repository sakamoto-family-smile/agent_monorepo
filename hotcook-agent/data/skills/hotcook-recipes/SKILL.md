# Skill: hotcook-recipes

ホットクック (シャープ KN-HW24H) でユーザーが調理可能なメニューを提案するための判断ルール集。
このスキルは agent モードの Claude が `menu-catalog.json` と一緒に読み込んで使う。

## 対象機種と前提

- **シャープ ヘルシオ ホットクック KN-HW24H** (容量 2.4L)
- 内蔵メニュー数: 145 (Phase 1 シードでは 30 件のみ収録)
- 無線 LAN 経由の COCORO KITCHEN 追加メニューは Phase 2 以降

## 提案の基本原則

1. **ユーザーの食材だけで作れるか** を最優先で判定する
   - 主材料 (`main_ingredients`) が 1 つでも足りなければ「不足あり」として根拠に明示
   - 任意材料 (`optional_ingredients`) は無くても提案可能

2. **調理時間とユーザーの状況を照合** する
   - リクエストに `max_cook_minutes` があれば必ず守る
   - 「予約調理希望」(`require_reservation=true`) のときは `reservation_ok=true` のメニューに限定

3. **まぜ技ユニットの要否** を明示する
   - `mixer_required=true` のメニューは予約調理対応外の機種環境もあるので注意喚起する

4. **詳細手順・分量は答えない**
   - 著作権配慮のため、本スキルとカタログには手順・分量を含まない
   - 「詳細は本体液晶 / COCORO KITCHEN アプリ / 取扱説明書 P.XX を参照してください」と必ず案内する

## ホットクックが活きる場面 / 活きない場面

| 向いている | 向いていない |
|---|---|
| 無水カレー、肉じゃが等の煮込み | 焼き目をつけたい料理 (ステーキ・餃子) |
| 蒸し鶏・蒸し野菜などのヘルシー調理 | フライ・天ぷら |
| 低温調理 (鶏ハム・ローストビーフ) | パリッと炒める系 (野菜炒め最終フェーズ) |
| 発酵 (ヨーグルト・甘酒) | 短時間で済む生もの・サラダ |
| 予約調理 (朝セット夜完成) | 麺の絶妙な茹で時間 |

ユーザーの食材や希望が「向いていない」側に寄っているときは、無理に内蔵メニューを推さず
「フライパン調理の方が向いています」と素直に案内すること。

## レスポンスのトーン

- 必ず日本語
- 候補ごとに **1〜2 行で根拠** (どの食材を消費する / 調理時間 / 予約可否) を述べる
- ランキング形式 (1 位〜)
- 末尾に「投資勧誘ではないが情報提供である」相当の **「実機の最新表示を優先してください」** ディスクレーマを付ける
