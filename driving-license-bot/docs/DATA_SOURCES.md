# データソース調達方針メモ（Phase 0 確定）

問題生成の事実根拠とする一次ソースの利用条件・取得方法・運用方針を整理する。
Phase 0 調査の結果、各データソースは原則 **政府標準利用規約 2.0（CC BY 4.0 互換）** または **公共データ利用規約 1.0（PDL1.0）** で利用可能であることを確認済み。**最終的な公開直前に最新利用規約を再確認すること**。

> 関連する設計上の位置付けは [DESIGN.md §2 データソース](./DESIGN.md#2-データソース) を参照。

## 1. 道路交通法・施行令・施行規則（e-Gov 法令検索）

### 概要

| 項目 | 内容 |
|---|---|
| 提供元 | デジタル庁（e-Gov 法令検索） |
| 取得手段 | e-Gov 法令API Version 2 |
| データ形式 | XML / JSON 簡易レスポンス（2026-03-15 追加） |
| 更新頻度 | 法令改正のタイミング（不定期、月次取得で十分） |
| ドキュメント | [Swagger UI](https://laws.e-gov.go.jp/api/2/swagger-ui) / [Redoc](https://laws.e-gov.go.jp/api/2/redoc/) |

### 利用条件（Phase 0 確認済み）

- e-Gov のコンテンツは **政府標準利用規約 第 2.0 版**（CC BY 4.0 互換）に準拠して利用可能。**商用利用も可能**。
- 出典の記載が必須。記載例: 「出典: e-Gov 法令検索（〇〇省）（URL）（最終確認日: YYYY-MM-DD）」
- コンテンツを編集・加工した場合は、出典とは別に編集・加工した旨を記載
- API のレート制限・認証要件の詳細は Swagger UI / Redoc / [利用規約](https://laws.e-gov.go.jp/terms/) を実装直前に再確認

### 取得対象

- 道路交通法（昭和35年法律第105号）
- 道路交通法施行令
- 道路交通法施行規則
- 道路標識、区画線及び道路標示に関する命令

### 運用方針

- 月次バッチ（[DESIGN.md §7](./DESIGN.md#7-法令改正追従パイプライン月次)）で全文を取得し GCS に snapshot 保存
- 前回 snapshot との diff を取り、変更条文に紐づく問題を `needs_review` にフラグ
- 問題スキーマの `sources[].url` には e-Gov の **条文単位アンカー付き URL** を格納
- 解説末尾に「出典: e-Gov 法令検索 [条文ページへ]」を必ず付与

## 2. 交通の方法に関する教則（警察庁）

### 概要

| 項目 | 内容 |
|---|---|
| 提供元 | 警察庁（国家公安委員会告示） |
| 根拠 | 道路交通法第 108 条の 28 第 4 項 |
| 取得手段 | 警察庁ウェブサイト掲載の PDF |
| 最新版（参考） | [2024-11-13 版 PDF](https://www.npa.go.jp/bureau/traffic/20241113kyousoku.pdf) |
| データ形式 | PDF（章節構造あり） |

### 利用条件（Phase 0 確認済み）

- 警察庁ウェブサイトは [利用規約](https://www.npa.go.jp/rules/index.html) で **公共データ利用規約 1.0（PDL1.0）** に準拠と明示。商用利用禁止の明示記載なし（=可）。
- 出典の記載が必須。記載例: 「出典: 警察庁ウェブサイト（当該ページの URL）」
- 編集・加工時は別途その旨を記載
- 教則 PDF への明示的な個別言及はないが、警察庁ウェブサイト全般が対象であるため適用される

### 構造化方針

PDF を直接 LLM に渡すと精度・コストが悪いため、Phase 0 で章節構造ありの JSON に変換する。

```
kyousoku-corpus.json
  - chapters:
    - chapter: 1
      title: "交通の方法 — 歩行者の心得"
      sections:
        - section: 1
          page: 5
          paragraphs:
            - id: "1-1-1"
              text: "..."
              page: 5
```

- PDF テキスト抽出 + 手動レビュー（運営者 1 人）で構造化
- 章節 ID と PDF ページ番号を保持し、解説からのリンクは `https://.../kyousoku.pdf#page=NN` で生成

### 引用範囲のガイドライン

著作権法第 32 条の引用要件（公表されたもの・引用の必然性・主従関係・明瞭区別・出典明示）を満たす範囲で原文を引用する。実務上は:

- 1 問あたりの引用は 1〜2 文以内（1 段落を超えない）
- 「教則によれば、…」のように引用部分を明確に区別
- 出典に教則のページ番号と最終確認日を必ず明記

## 3. 道路標識・標示・信号機

### 概要

道路標識、区画線及び道路標示に関する命令で定められた様式に基づく標識データ。

### 著作権の整理（Phase 0 確認済み）

- 道路標識のデザインは「道路標識、区画線及び道路標示に関する命令」の様式に基づき、国家機関の告示として **著作権法第 13 条により著作権の対象とならない**（パブリックドメイン）
- Wikimedia Commons には [Japan road sign 115](https://commons.wikimedia.org/wiki/File:Japan_road_sign_115.svg) など多数の SVG が PD として公開されている
- ただし一部の SVG は埋め込みフォント（例: GD-HighwayGothicJA）の関係で **CC-BY-SA 2.1 JP** が要求されるケースがある → 利用前に各ファイルのライセンスを個別確認

### 調達方針（Phase 0 確定）

**ハイブリッド方針**:

1. **第一選択**: Wikimedia Commons の Public Domain SVG（フォント縛りのないもの）を優先利用
2. **第二選択**: フォント縛り CC-BY-SA があるものは「出典記載 + 改変表示」で利用、または PD 版を別途確保
3. **第三選択（最終手段）**: PD 版が見つからない標識のみ、運営者が SVG を新規作成（公式様式に基づく）

すべての標識画像は GCS の `signs/` 配下に保存し、メタデータに **ライセンス種別と原典 URL** を必ず記録する。

### 標識カタログ JSON

```yaml
signs-catalog.json:
  - id: "C-301"          # 規制標識 / 一時停止
    title: "一時停止"
    category: "regulatory"
    image_uri: "gs://.../signs/C-301.svg"
    license: "PD"        # PD | CC-BY-SA-2.1-JP | self-made
    source_url: "https://commons.wikimedia.org/wiki/File:Japan_road_sign_330.svg"
    description: "..."
    legal_basis:
      - law_id: "335M50000003003"
        article_or_form: "別表第二"
```

## 6. 学科試験の公式合格基準・配点（Phase 0 確認済み）

設計書 [DESIGN.md §9.3](./DESIGN.md#93-模擬試験モード) と整合させる。

### 仮免学科試験

- 問題数: **50 問**
- 配点: **各 2 点**
- 満点: **100 点**
- 試験時間: **30 分**
- 合格点: **90 点以上**

### 本免学科試験

- 問題数: **95 問**（〇× 90 問 + イラスト問題 5 問）
- 配点:
  - 〇× 問題: 各 1 点（90 点分）
  - イラスト問題: 各 2 点（10 点分） — 1 イラストにつき 3 問の判定があり、**3 問すべて正解で 2 点**
- 満点: **100 点**
- 試験時間: **50 分**
- 合格点: **90 点以上**

### 本サービスでの実装方針

- **Phase 1〜5**: 〇× 問題のみで 100 点換算（90 点合格）に揃える。設計書 §9.3 の暫定基準で運用
- **Phase 6 以降**: イラスト問題（3 連問形式）を別 format として追加し、本試験を完全再現

### 出典

- [シカクン: 学科試験の合格点](https://www.shikakun.net/department_of_item/passing_grade/)
- [らくらく免許くらぶ: 学科試験](https://rakuraku-menkyo.jp/exam/gakka-scoreup/)

## 7. データ取得バッチ全体像（Phase 4+）

```
Cloud Scheduler (月初)
  ├─ e-Gov API → GCS: law-snapshots/YYYY-MM/*.xml
  ├─ 警察庁教則 PDF → 手動 or 半自動更新（変更検知 + 運営者通知）
  └─ 標識カタログ → 変更稀少のため手動更新
        │
        ▼
  diff 検出 → 影響問題を needs_review にフラグ
```

## 9. 出典表記の標準フォーマット（本サービス共通）

LINE Bot の解説 Flex Message 末尾、および利用規約・プライバシーポリシーで以下のフォーマットを使用：

### 法令引用

```
出典: e-Gov 法令検索 道路交通法 第○条 第○項
URL: https://elaws.e-gov.go.jp/document?lawid=...#Mp-At_36
最終確認日: YYYY-MM-DD
```

### 教則引用

```
出典: 警察庁「交通の方法に関する教則」第○章 第○節（PDF p.NN）
URL: https://www.npa.go.jp/.../kyousoku.pdf#page=NN
最終確認日: YYYY-MM-DD
```

### 標識画像

メタデータに license 区分（PD / CC-BY-SA-2.1-JP / self-made）を保持し、CC-BY-SA 由来の画像を表示する場面では「出典: Wikimedia Commons (CC-BY-SA-2.1-JP)」を画像下に併記。

### 編集・加工の表示

問題本文・解説は AI が生成し運営者が編集しているため、設計書 §0.2 の定型句で「AI により自動生成された」旨を必ず明示する（編集・加工の表示要件をカバー）。
